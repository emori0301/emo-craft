# Slack絵文字ジェネレータアプリ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
Slackで使用できるカスタム絵文字を簡単に作成できるWebアプリケーションを開発する。ユーザーが文字サイズ、太さ、フォント、カラーなどを自由にカスタマイズして、オリジナルの絵文字を生成できる。

### 1.2 技術スタック

#### コア技術
- **フロントエンド**: Next.js 15+ (App Router, React Server Components)
- **バックエンド**: Next.js API Routes + tRPC v11
- **データベース**: PostgreSQL (Prisma ORM)
- **認証**: Better Auth - モダンな認証ライブラリ（OAuth + Credentials認証）
- **UIコンポーネント**: shadcn/ui
- **スタイリング**: Tailwind CSS v4
- *
#### データフェッチング・状態管理
- **tRPC**: 型安全なAPI通信
- **TanStack Query (React Query)**: サーバー状態管理、キャッシング
- **Zustand**: クライアント側のグローバル状態管理
- **React Hook Form**: フォーム状態管理とバリデーション

#### バリデーション・フォーム
- **Zod**: スキーマバリデーション（tRPC、React Hook Form、Better Authと統合）
  - APIリクエスト/レスポンスのバリデーション
  - フォーム入力のバリデーション
  - 環境変数のバリデーション
- **React Hook Form**: パフォーマンスの高いフォーム管理（Zodと統合）
- **@hookform/resolvers**: React Hook FormとZodの統合

#### 開発ツール
- **Biome**: 高速なリンター・フォーマッター（ESLint + Prettierの代替）
- **TypeScript**: 型安全性
- **Prisma**: ORMとマイグレーション管理

#### 画像処理
- **HTML5 Canvas API**: クライアントサイド画像生成
- **Canvas 2D Context**: ドット絵エディタの描画処理
- **ImageData API**: ピクセル単位の操作
- **Sharp**: サーバーサイド画像処理（将来実装）

#### フォント管理
- **next/font**: Next.jsのフォント最適化
- **Google Fonts API**: 外部フォント読み込み

#### 開発環境・コンテナ
- **Docker**: 開発環境のコンテナ化
- **Docker Compose**: マルチコンテナの管理
- **PostgreSQLコンテナ**: 開発用データベース
- **開発環境の統一**: チーム全体で同じ環境を共有

#### デプロイ・インフラ
- **開発環境**: Docker Composeでローカル環境を構築
- **本番環境**: 
  - **Vercel**: アプリケーションのホスティング（推奨）
  - **Vercel Postgres** または **Supabase**: PostgreSQLホスティング
  - **Vercel Blob Storage**: 画像ストレージ（本番環境）
- **Dockerイメージ**: 必要に応じて本番環境でもDockerを使用可能

#### その他
- **Lucide React**: アイコンライブラリ（shadcn/uiで使用）
- **date-fns**: 日付処理
- **clsx / tailwind-merge**: クラス名管理（cn()ユーティリティ）
- **framer-motion**: アニメーション（オプション、shadcn公式サイトのようなスムーズなアニメーション用）
- **radix-ui**: shadcn/uiの基盤となるUIプリミティブ（自動インストール）

## 2. 機能要件

### 2.1 基本機能（MVP）

#### 2.1.1 絵文字生成機能
- **文字入力**
  - ユーザーが絵文字にしたい文字を入力できる
  - 複数文字の入力に対応（最大推奨：2-3文字）
  - リアルタイムプレビュー表示

- **文字サイズ調整**
  - スライダーまたは数値入力でサイズを調整可能
  - 範囲：24px ～ 512px
  - デフォルト：128px

- **太さ（Font Weight）調整**
  - 選択可能な太さ：100, 200, 300, 400, 500, 600, 700, 800, 900
  - デフォルト：400 (Normal)

- **フォント選択**
  - システムフォント + Google Fontsから選択可能
  - 推奨フォント：
    - Noto Sans JP（日本語対応）
    - Inter
    - Roboto
    - Open Sans
    - その他、絵文字に適したフォント
  - デフォルト：Noto Sans JP

- **カラー選択**
  - カラーピッカーで背景色と文字色を個別に設定可能
  - 背景色：デフォルト透明（透明選択可能）
  - 文字色：デフォルト黒
  - プリセットカラーパレットも提供

- **キャンバスサイズ**
  - 固定サイズ：128x128px（Slack絵文字標準サイズ）
  - 将来的に256x256px、512x512pxにも対応可能

#### 2.1.2 ドット絵エディタ機能
- **エディタモード選択**
  - テキストベースエディタとドット絵エディタを切り替え可能
  - タブまたはボタンでモードを選択

- **キャンバスサイズ選択**
  - 16x16、32x32、64x64、128x128ピクセルから選択可能
  - デフォルト：32x32（ドット絵に適したサイズ）
  - カスタムサイズも設定可能（最大128x128）

- **グリッド表示**
  - ピクセルグリッドの表示/非表示を切り替え可能
  - グリッドの色と透明度を調整可能
  - 拡大表示時のグリッド表示

- **描画ツール**
  - **ペンツール**: ピクセル単位で描画
  - **消しゴムツール**: ピクセルを削除（透明化）
  - **バケツツール**: 同じ色の連続領域を塗りつぶし
  - **スポイトツール**: キャンバス上の色を取得
  - **直線ツール**: 直線を描画
  - **矩形ツール**: 四角形を描画（塗りつぶし/枠線）
  - **円形ツール**: 円を描画（塗りつぶし/枠線）

- **カラーパレット**
  - カラーピッカーで色を選択
  - カスタムカラーパレットの保存
  - よく使う色の履歴表示
  - 透明度（アルファチャンネル）の設定
  - プリセットカラーパレット（ドット絵向けの色）

- **拡大・縮小表示**
  - ズームイン/アウト機能（100%～800%）
  - ピクセルパーフェクト表示
  - 全体表示と拡大表示の切り替え

- **編集機能**
  - **アンドゥ/リドゥ**: 操作の取り消しとやり直し（最大50ステップ）
  - **全消去**: キャンバスをクリア
  - **コピー/ペースト**: 選択範囲のコピーとペースト
  - **左右反転**: 水平方向に反転
  - **上下反転**: 垂直方向に反転
  - **回転**: 90度単位で回転

- **レイヤー機能**（将来実装）
  - 複数レイヤーの作成と管理
  - レイヤーの表示/非表示
  - レイヤーの順序変更
  - レイヤーの統合

- **アニメーション機能**（将来実装）
  - フレーム単位での編集
  - フレームの追加・削除・複製
  - フレーム間のコピー
  - アニメーション速度の調整
  - プレビュー機能

#### 2.1.3 プレビュー機能
- リアルタイムで変更内容を反映
- 生成された絵文字のプレビュー表示
- 複数のサイズでのプレビュー（128px, 256px, 512px）

#### 2.1.4 エクスポート機能
- PNG形式でのダウンロード
- 解像度選択可能（128px, 256px, 512px）
- ファイル名は自動生成（例：emoji_文字_タイムスタンプ.png）

#### 2.1.5 保存機能（ユーザーアカウント機能）
- ユーザー登録・ログイン機能
- 作成した絵文字を保存
- 保存した絵文字の一覧表示
- 保存した絵文字の編集・削除
- 保存した絵文字の再ダウンロード

### 2.2 将来の拡張機能（Phase 2以降）

#### 2.2.1 アニメーション機能
- GIF形式でのエクスポート
- アニメーション効果の選択：
  - フェードイン/アウト
  - 回転
  - 拡大/縮小
  - バウンス
  - カスタムアニメーション
- アニメーション速度の調整
- フレーム数の設定

#### 2.2.2 高度なカスタマイズ
- グラデーション背景
- 影（ドロップシャドウ）の追加
- 枠線の追加
- 複数レイヤーの組み合わせ
- アイコン・シンボルの追加

#### 2.2.3 共有機能
- 生成した絵文字のURL共有
- 公開/非公開設定
- 他のユーザーの絵文字を閲覧・コピー

#### 2.2.4 テンプレート機能
- よく使われる絵文字のテンプレート
- カテゴリ別テンプレート（感情、アクション、記号など）

## 3. 非機能要件

### 3.1 パフォーマンス
- プレビューの更新はリアルタイム（遅延 < 100ms）
- 画像生成処理は2秒以内に完了
- ページ読み込み時間は3秒以内

### 3.2 ユーザビリティ
- 直感的なUI/UX
- モバイルレスポンシブ対応（基本的な操作は可能）
- キーボードショートカット対応

### 3.3 セキュリティ
- ユーザー認証はBetter Authで実装
- OAuth認証（Google, GitHub, Discord等）に対応
- Credentials認証（メール・パスワード）にも対応
- 入力値のバリデーション（Zodスキーマ）
- CSRF保護
- セッション管理の適切な実装
- ファイルアップロードの検証とサニタイゼーション

### 3.4 ブラウザ対応
- Chrome（最新版）
- Firefox（最新版）
- Safari（最新版）
- Edge（最新版）

## 5. データベース設計

### 5.1 テーブル構成

#### User（ユーザー）
- id: String (UUID)
- email: String (unique)
- emailVerified: Boolean (メール認証済みフラグ)
- name: String
- image: String? (プロフィール画像URL)
- createdAt: DateTime
- updatedAt: DateTime

#### Account（OAuthアカウント / Better Auth）
- id: String (UUID)
- userId: String (User.idへの外部キー)
- accountId: String (プロバイダー側のアカウントID)
- providerId: String (google, github等)
- accessToken: String?
- refreshToken: String?
- expiresAt: DateTime?
- createdAt: DateTime
- updatedAt: DateTime

#### Session（セッション / Better Auth）
- id: String (UUID)
- userId: String (User.idへの外部キー)
- expiresAt: DateTime
- token: String (unique)
- ipAddress: String?
- userAgent: String?
- createdAt: DateTime
- updatedAt: DateTime

#### Verification（メール認証用 / Better Auth）
- id: String (UUID)
- identifier: String (メールアドレス)
- value: String (認証トークン)
- expiresAt: DateTime
- createdAt: DateTime

#### Emoji（絵文字）
- id: String (UUID)
- userId: String (User.idへの外部キー)
- name: String (絵文字名)
- editorType: String (Enum: "text" | "pixel") - エディタタイプ
- text: String? (テキストエディタ時の入力文字、nullの場合はドット絵)
- fontSize: Int? (テキストエディタ時のみ)
- fontWeight: Int? (テキストエディタ時のみ)
- fontFamily: String? (テキストエディタ時のみ)
- backgroundColor: String? (HEXカラー、nullの場合は透明)
- textColor: String? (テキストエディタ時の文字色)
- canvasSize: Int (キャンバスサイズ、デフォルト128)
- pixelData: Json? (ドット絵エディタ時のピクセルデータ、ImageData形式)
- pixelCanvasSize: Int? (ドット絵エディタ時のキャンバスサイズ、16/32/64/128)
- imageUrl: String? (生成された画像のURL)
- isPublic: Boolean (デフォルトfalse)
- createdAt: DateTime
- updatedAt: DateTime

#### EmojiTemplate（テンプレート、将来実装）
- id: String (UUID)
- name: String
- category: String
- settings: Json (絵文字設定のJSON)
- createdAt: DateTime

  - 履歴管理（アンドゥ/リドゥ）の実装
- **サーバーサイド（将来）**: SharpまたはCanvas（node-canvas）を使用

### 8.2 フォント管理
- **next/font**: Next.jsのフォント最適化を使用

- 必要なフォントを事前に読み込む
- フォントの動的読み込みに対応（絵文字生成時）

### 8.3 状態管理
- **サーバー状態**: TanStack Query + tRPC（自動キャッシング、再フェッチ）
- **クライアント状態**: 
  - ローカル状態: React Hook（useState, useReducer）
  - グローバル状態: Zustand（エディタ設定、UI状態など）
- **フォーム状態**: React Hook Form + Zodバリデーション

### 8.4 ファイルストレージ
- **開発環境**: 生成した画像をBase64でDBに保存（小規模テスト用）
- **本番環境**: Vercel Blob StorageまたはSupabase Storageを使用
- 画像URLをDBに保存し、実際のファイルはストレージに保存

### 8.5 Docker環境

#### 8.5.1 Docker構成
- **docker-compose.yml**: マルチコンテナの定義
  - Next.jsアプリケーションコンテナ
  - PostgreSQLコンテナ（開発用）
  - 必要に応じてRedisコンテナ（セッション管理用、オプション）
- **Dockerfile**: Next.jsアプリケーションのビルドと実行
- **.dockerignore**: 不要なファイルをビルドコンテキストから除外

#### 8.5.2 開発環境のセットアップ
- **起動コマンド**: `docker-compose up`
- **データベース接続**: コンテナ間通信でPostgreSQLに接続
- **ホットリロード**: 開発時のコード変更を即座に反映
- **ボリュームマウント**: ソースコードをマウントして開発効率を向上

#### 8.5.3 PostgreSQLコンテナ設定
- **イメージ**: postgres:16-alpine（軽量版）
- **環境変数**: 
  - POSTGRES_USER
  - POSTGRES_PASSWORD
  - POSTGRES_DB
- **データ永続化**: Dockerボリュームでデータを保持
- **ポート**: 5432（ホストマシンからもアクセス可能）

#### 8.5.4 開発ワークフロー
1. `docker-compose up -d` でコンテナ起動
2. `docker-compose exec app npm run dev` で開発サーバー起動
3. Prismaマイグレーション: `docker-compose exec app npx prisma migrate dev`
4. データベース確認: `docker-compose exec db psql -U <user> -d <database>`
5. `docker-compose down` でコンテナ停止

#### 8.5.5 本番環境との差分
- **開発環境**: Docker Composeでローカル環境を構築
- **本番環境**: Vercelにデプロイ（Vercel Postgresを使用）
- 環境変数で接続先を切り替え可能

### 8.6 環境変数管理

#### 8.6.1 環境変数の種類
- **データベース接続**: DATABASE_URL（開発/本番で異なる）
- **認証**: Better Auth関連のシークレット
- **OAuth**: Google、GitHub等のクライアントID/シークレット
- **アプリケーション**: NEXTAUTH_URL、APP_URL等

#### 8.6.2 環境変数の管理方法
- **開発環境**: `.env.local` ファイル（gitignoreに追加）
- **Docker環境**: `docker-compose.yml`の`environment`セクションまたは`.env`ファイル
- **本番環境**: Vercelの環境変数設定
- **バリデーション**: Zodスキーマで環境変数を検証（`env.mjs`等）

#### 8.6.3 Docker環境での環境変数
- `docker-compose.yml`で環境変数を定義
- `.env`ファイルから読み込み（docker-composeが自動的に読み込む）
- コンテナ間で環境変数を共有
